## 输出页质量门槛

从 2026-04-29 起，凡是新增或重写 output 型导览页，不能只检查"链接能点"，还要检查"点进去有没有内容"。最低规则：

1. output 页面直接链接到的 redology / poetry / concepts / events / characters 页面，不能只有一两句占位说明。
2. redology 页面至少要说明：核心问题、主要观点分歧、证据层级、相关原文/章节、Wiki 处理规则。
3. poetry 页面至少要说明：文本位置、叙事功能、人物/主题关系、可回读的章节入口。
4. events 页面至少要说明：事件经过、叙事功能、人物关系变化、原文锚点。
5. 如果子页面暂时达不到质量门槛，output 中不要把它包装成成熟入口；要么先扩写，要么标为"待补"。

## 总体路线：从"可读 Wiki"走向"研究型 Wiki"

后续不建议再大规模生成新页面。更好的路径是：

1. 以现有人物、概念、事件、地点为切入点；
2. 搜集权威材料；
3. 在原节点补"研究视角"和 `sources:`；
4. 必要时新增 redology 专题；
5. 从专题反向回链章节、人物、事件、概念；
6. 每轮后运行链接和占位检查。

这样 Wiki 会越来越像网状知识库，而不是一堆孤立读书笔记。
